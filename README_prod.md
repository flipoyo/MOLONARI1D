README détaillant les instructions et exigeances de l'équipe responsable de la production

Chaque membre de l'équipe de production doit avoir une bonne vision globale du projet et en particulier une vision très spécifique de son groupe : il faut se tenir à jour des avancées et objectifs à suivre de chaque sous-groupe éventuel du groupe.
Ces objectifs doivent être correctement défini afin de rester dans le cadre du projet et de satisfaire les requirements de celui-ci et d'éventuellement proposer des pistes d'améliorations.

Il faut être capable de présenter ces avancées et ces objectifs chaque jour lors des réunions de l'équipe avec M. Flipo (et donc bien évidemment d'être investi en participant à ces réunions).

Un autre point clé pour le responsable production est la gestion du Github du projet. 
Les principales étapes de cette gestion sont rappelées ci dessous: 

 1. Créer une branche Année - Nom du groupe dans laquelle seront remontées les modifications respectives des sous-groupes du groupe 
 2. Créer des issues et des branches correspondantes et expliquer aux membres de son groupe ce fonctionnement pour que chacun puisse travailler sur la branche et sur l'issue correspondantes à sa tâche. 
 3. Expliquer aux membres du groupe le principe de pull request qui doit être réalisée afin de demander au responsable production d'ajouter leur travail sur la branche du groupe (Branche Année-Nom du groupe)
 4. Pour chaque pull request : passer en revue les modifications faites par la personne qui demande la pull request et décider d'accepter ou non les modifications proposées en commentant les raisons de ce choix. (En expliquant brièvement pourquoi elles ont été acceptées ou ce qu'il faudrait changer pour qu'elles le soient)
 5. Expliquer aux membres de son groupe de se tenir à jour vis-à-vis des branches supérieures (Main et Année-Nom du groupe) via la commande git pull origin Nom de la branche dont on veut récupérer tous les changements ajoutées
 6. Faire des pull requests pour ajouter sur la branche Main les modifications précédemment acceptées dans la branche Computing et en informer M. Flipo.
